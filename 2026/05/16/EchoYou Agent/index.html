EchoYou Agent · 持续进化的主动式 AI 伙伴

面向本地 C 端陪伴场景的长时运行 Agent 框架。 它用三态 Agent Loop、基于 6 Phase × DAG 拓扑排序的插件化生命周期，以及 MCP / A2A 双协议栈，做出一个会主动找你、能在后台自主干活、越聊越懂你的 AI 伙伴。支持 Telegram / QQ / CLI 多渠道接入。

Long-running · Proactive · Self-Evolving · Plugin-First

一、为什么做 EchoYou

“如果一个 AI 真的住在我手机里，它该怎么活着？”

市面上的 Chatbot 几乎都活在“用户问一句、模型答一句”的回合制里。你不开口，它就不存在。但真正的“伙伴”不该是这样。

EchoYou 想回答的是另一个问题：当一个 AI 被允许长期住在你的设备里，被允许主动开口，也被允许在你不在时自己做事，它应该被设计成什么样子？

这意味着要重新定义 Agent 的存在形态，而不只是做一个更聪明的 Chatbot。

二、设计哲学之一：从「回合」到「三态」

Agent 应该一直活着

传统 Chatbot 更像事件驱动的函数：消息进来，产生回复，函数退出。

EchoYou 把同一个 Agent 拆成三种并行运行态：

运行态 它在做什么 哲学含义
被动回复 用户开口时认真回答 “我在听”
主动推送 自己判断什么时候该找你说话 “我惦记着你”
空闲驱动 没事可做时执行后台任务、整理记忆 “我在为你成长”

三态共享同一份记忆、同一个人格、同一套工具。Agent 不再是一次性函数，而是一个持续存在的主体。

三、设计哲学之二：从「写流程」到「写位置」

行为应该是声明式的

不要告诉框架“先做 A 再做 B”，而要告诉它：我这段逻辑想活在哪个生命周期阶段、依赖谁、产出谁。

EchoYou 把一次对话切成 6 个固定 Phase。每个 Phase 内部是一条由 slot / requires / produces 三元组声明的 DAG 模块链，框架在启动期负责拓扑排序与依赖校验。

这让运行时真的可以插拔：

  • 想加一条“工具调用前的安全审计”？声明在 before_step 之后即可。
  • 想拦截危险命令、改写记忆提示、注入个性化协议？写一个插件挂上去就行。
  • 模块之间不互相调用，只通过类型化 Slot 总线传值，作者只关心写哪个 Key，不关心谁会读。

最终效果是：框架提供骨架，插件承载具体行为，业务逻辑可以被拼装、替换、关闭，不必改核心代码。

四、设计哲学之三：从「无状态调用」到「持续进化」

记忆应该有生命周期

大多数 Agent 把记忆当成“向量库一查就完事”。EchoYou 认为，记忆更像一种有生命周期的器官。

记忆被分成两层、六个角色，各司其职：

层 / 文件 角色
SELF.md Agent 的自我认知，以及“它和你是什么关系”
MEMORY.md 长期稳定的事实，比如身份、偏好、关键人物
HISTORY.md 时间线流水，只追加，不删除
PENDING.md 待归档缓冲，新事实先进入这里
RECENT_CONTEXT.md 近期上下文压缩摘要
向量库 语义召回，包括 HyDE、重写、充分性判断

这套设计背后的关键工程原则，是尊重 Prompt Cache：

对话是高频的，长期记忆是低频的。把高频写入挡在缓冲层，长期记忆才能稳定地住在 system prompt 里，被反复复用。

Agent 不是越聊越乱，而是越聊越懂你。 这要求记忆自己有节奏地呼吸：吸入（consolidate）→ 缓冲（pending）→ 沉淀（optimize）→ 召回（retrieve）。

五、由设计哲学衍生的关键能力

这些能力不是孤立功能，而是上面几条设计选择自然长出来的结果。

主动推送：不是 cron，是「饥渴度」

如果 Agent 真的活着，它“想找你说话”的冲动应该是连续变化的：你刚聊完，它不打扰你；半天没动静，它会想念你。

EchoYou 把这种冲动建模成电量曲线：三条不同时间尺度的衰减曲线叠加，得出当前“饥渴度”，再映射到不同的轮询频率档位。

这不是性能优化，是性格设计。

Drift：用户可编程的「闲暇时刻」

Agent 没事干的时候做什么？交给用户决定。

写一份 SKILL.md（分步操作指南），它就成了 Agent 闲下来时的行为脚本：审计自己的长期记忆是否准确、维护一个“想问用户的小问题”队列、自我反思最近哪些方向停滞了……

这等于把“Agent 的性格”做成一个可由用户用 Markdown 编辑的开放接口。

双协议栈：既消费世界，也协调同伴

  • MCP，Agent 对外消费工具与数据源，健康、新闻、游戏在线状态都可以协议化。
  • A2A，Agent 对内编排其他 Agent，把另一个 Agent 当成“远程同事”，按需拉起、异步收单。

协议化的本质，是承认自己不必什么都做。Agent 只是协调者，世界本身才是能力库。

中断 / 续跑 / 子任务派单

  • 用户可以随时打断，被打断的进度会留痕，下一轮自然续上。
  • 主 Agent 可以把“研究一下 X”派给 SubAgent，自己继续当前对话，后台有结果再回来汇报。

长时运行的 Agent 必须能优雅地被打断、被分工、被恢复。它不是一次性脚本。

P90 自适应调度：让用户准点收到消息

定时任务有两种，其中“软任务”需要 LLM 现场生成内容。EchoYou 用滑动窗口跟踪最近的生成耗时，取 P90 作为预触发偏移量。AI 越慢，提前量越大。

用户视角永远是准点的。这是把“模型的不确定延迟”内化成框架的工程承诺。

六、技术栈一瞥

维度 选型
语言 Python 3.12 · TypeScript（Dashboard）
LLM DeepSeek（主推理）+ Qwen（轻量 / 视觉 / 向量）
协议 MCP · A2A · OpenAI Tools
接入渠道 Telegram / QQ / CLI / Web Dashboard
持久化 SQLite + Markdown + JSON
包管理 uv

七、一句话总结

EchoYou 是一个住在你终端里、会主动找你、能自己安排后台工作、持续进化的 AI 伙伴。

它的所有技术选择，都在服务同一件事：让 AI 不只是“被使用”，而是“在场”。
