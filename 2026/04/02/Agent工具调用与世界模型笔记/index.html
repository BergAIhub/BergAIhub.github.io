Agent工具调用与世界模型笔记

工具不是外挂。工具调用失败的根因，常常是 Agent 对“世界如何运作”的内部模型不完整。

先说结论

Agent 能不能用好工具，不在于工具数量，而在于它是否知道“何时调用、调用后会发生什么、失败后怎么恢复”。

真正的工具能力，本质是世界模型能力。

工具只是把这个模型投射到现实环境的接口。

先把基础知识补齐

所谓世界模型，可以理解为 Agent 对外部环境的可执行认知地图。

它至少包含四类信息：

  • 状态结构：环境里有哪些对象，彼此关系是什么。
  • 作用规则：某个操作会对状态造成什么影响。
  • 观测机制：哪些结果可见，哪些结果需要间接推断。
  • 不确定性：哪些部分经常失败，失败后有哪些替代路径。

没有这四类信息，调用工具就只是“盲按按钮”。

这个问题到底在讨论什么

很多团队会把问题描述成“function calling 准确率不高”。

这只是表层。

更深层的问题是：Agent 是否具备足够的因果理解去驱动工具。

比如“调用搜索 API”这件事，本质不只是发请求。

它还包含查询构造、结果筛选、证据融合、冲突处理、更新结论。

如果这些环节没有建模，调用成功也没有意义。

从基础往外延伸

成熟的工具调用链一般分五步：

  1. 意图解析：当前子目标到底是什么。
  2. 工具选择：哪个工具最可能提供有效增量信息。
  3. 参数构造：参数是否足够具体、可执行、可复现。
  4. 结果解释：工具返回值如何映射回任务状态。
  5. 策略更新：下一步继续调用、切换工具还是请求人工输入。

其中最容易被忽略的是第 4 步。

很多系统把返回值直接拼到上下文，默认模型会自动理解。

这导致“拿到很多数据，但没有形成决策”。

这个问题为什么容易想错

第一个误区是“多接几个工具，能力就会线性增长”。

现实里工具越多，路由复杂度和错误传播路径也越多。

第二个误区是“工具失败就重试”。

重试可以处理偶发故障，不能处理语义级错误。

如果参数本身错了，重试只会放大成本。

第三个误区是“工具结果一定可信”。

工具只是数据通道，不是事实担保。

返回成功状态码，不代表业务逻辑正确。

回到第一性原理

工具调用系统的底层约束有三个：

  • 观测成本：每一次调用都消耗时间和资源。
  • 语义损失：自然语言意图映射到结构化参数时会失真。
  • 外部噪声：API 波动、数据延迟、权限限制会持续存在。

这意味着最优策略不是“尽量多调用”，而是“用最少调用获得最大决策增量”。

真正的机制是什么

高质量 Agent 会把工具当作“假设检验器”。

它不是为了证明自己对，而是为了快速排除错误路径。

典型机制是：

  • 先形成可检验假设。
  • 选择最便宜、信息增量最高的调用验证假设。
  • 基于结果更新假设置信度。
  • 当置信度不足时主动补充证据，而不是强行给结论。

这套机制让系统从“调用导向”变成“决策导向”。

用这个模型重新看问题

一个看似聪明但经常翻车的 Agent，通常会出现这些信号：

  • 工具调用日志很长，但任务状态几乎不推进。
  • 相同失败模式反复出现，没有形成防错策略。
  • 结果解释不一致，上一轮结论和下一轮策略断裂。

如果出现这些信号，优先改世界模型，不要先换模型。

我现在怎么判断

评估工具链是否健康，我会看这五点：

  1. 每次调用前是否有清晰的“要验证什么”。
  2. 返回结果是否被结构化解释，而不是原样堆入上下文。
  3. 失败类型是否被分类处理，而不只是统一重试。
  4. 调用策略是否体现了信息增量优先，而不是工具偏好。
  5. 日志是否足够让人复盘“为何做出这一步决策”。

总结

工具能力从来不是接口能力，而是认知能力。

没有世界模型，工具越多越乱。

有了世界模型，少量工具也能打出高质量决策链。

深度思考

工具调用的上限，取决于 Agent 对世界因果结构的理解，而不是 API 列表长度。很多失败案例看起来是参数填错，本质却是世界模型残缺：它不知道这个工具改变了什么状态，也不知道结果该如何反映回任务判断。没有因果意识，调用就会退化成“试一试”，重试越多成本越高。更深层的实践经验是，把每次调用都当成一次假设检验，而不是信息收集。先明确要证伪什么，再选择信息增量最大的工具，最后根据返回结果更新置信度。这样做会让调用次数下降，但决策质量上升。另一个关键是负证据管理：当结果与预期冲突时，系统能否主动推翻旧路径。多数 Agent 不是缺少成功路径，而是缺少放弃错误路径的机制。补上这一点，工具能力会出现质变。

最后给一个可执行标尺：把这套方法交给一个不在场的同事，他能否仅凭日志、配置和文档，在一小时内复盘决策并复现实验结果。如果做不到，说明系统仍依赖个人经验而不是工程能力。继续把隐性判断显式化，直到可测、可审计、可回滚。
