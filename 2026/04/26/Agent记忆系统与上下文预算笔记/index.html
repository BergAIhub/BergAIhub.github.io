Agent记忆系统与上下文预算笔记

记忆不是“存得越多越好”，而是“在有限上下文里，保留最能改变决策的那部分信息”。

先说结论

大多数 Agent 的退化不是因为遗忘，而是因为记忆污染。

把不关键的信息留在上下文里，会直接挤压关键证据，导致推理链变形。

记忆系统的核心任务是压缩信息熵，不是堆积日志。

先把基础知识补齐

Agent 的记忆通常分三层：

  • 工作记忆：当前回合即时可用信息，容量最小、时效最强。
  • 情节记忆：任务过程中的关键事件和决策轨迹。
  • 语义记忆：跨任务稳定知识，如规则、偏好、长期事实。

这三层不是按存储介质分，而是按“对决策时效的贡献”分。

如果不分层，系统会把短期噪声和长期规则混在一起。

这个问题到底在讨论什么

我们讨论的不是“上下文窗口够不够大”。

窗口再大也有边界，而且成本会随长度上升。

真正的问题是：哪些信息应该进入当前决策上下文，哪些应该被摘要、索引、或暂时丢弃。

这本质是一个预算分配问题。

从基础往外延伸

在工程上，常见策略是“写入前过滤 + 读取前排序”。

写入前过滤决定什么值得记住。

读取前排序决定什么先被模型看到。

过滤可以基于四个信号：

  • 任务相关性：是否直接影响当前目标。
  • 证据强度：是否有可验证来源。
  • 新颖性：是否补充了此前没有的信息。
  • 可复用性：是否会影响后续多步决策。

排序可以基于三个维度：

  • 近期性：最新状态优先。
  • 重要性：高影响信息优先。
  • 冲突性：与当前结论冲突的信息优先审阅。

冲突性优先是很多系统忽视的关键点。

如果只按相似度召回，模型会越来越相信自己原有判断，形成确认偏差。

这个问题为什么容易想错

误区一是“把所有中间过程都喂给模型更安全”。

实际上这会增加干扰项，让模型难以定位信号。

误区二是“摘要一次就够了”。

摘要本身也会丢信息，且可能引入摘要者偏见。

没有版本化摘要和可追溯原文，错误会被长期固化。

误区三是“检索召回越多越好”。

召回太多会稀释关键证据，造成注意力竞争。

回到第一性原理

记忆系统受两个根本约束：

  • 注意力是稀缺资源：模型不能同时高质量处理过多互相竞争的信息。
  • 存储不是理解：存下信息不等于形成可用于决策的结构。

所以好记忆系统必须服务决策，而不是服务归档。

真正的机制是什么

有效记忆系统通常具备以下闭环：

  1. 采集：记录原始事件与证据来源。
  2. 归一：提取结构化字段，如结论、置信度、适用条件。
  3. 分层：落到工作/情节/语义不同层级。
  4. 检索：按当前任务目标做条件召回。
  5. 反证：主动检查召回结果里是否存在冲突信号。
  6. 更新：根据新证据提升、降级或撤销旧记忆。

没有第 5 和第 6 步，记忆会变成“只增不减”的僵化系统。

用这个模型重新看问题

当 Agent 出现“同样错误反复犯”，不要先怪模型。

先查记忆系统是否有这三类问题：

  • 错误结论被写进长期记忆，且没有撤销机制。
  • 新证据进入后，没有触发旧记忆置信度重估。
  • 检索层只看语义相似，不看时间和冲突关系。

这三类问题会把系统拖进路径依赖。

我现在怎么判断

我会用一个简化检查表：

  1. 每条长期记忆是否带来源、时间、置信度。
  2. 摘要是否可回溯到原始证据。
  3. 检索策略是否支持反证优先。
  4. 是否存在记忆撤销或降级机制。
  5. 上下文预算是否显式分配给“目标、证据、反例、待确认项”。

总结

记忆系统不是知识仓库，而是决策引擎的前置筛选器。

上下文预算的本质是注意力预算。

谁能在预算内保留高价值证据，谁的 Agent 就更稳。

深度思考

记忆系统最容易被误解成“越多越好”，但真实工程里，信息过载比信息缺失更致命。上下文窗口不是仓库，而是注意力战场，任何低价值历史都会和关键证据抢位置。深度问题在于，很多系统只做写入，不做遗忘策略，结果把噪声永久固化。真正成熟的记忆设计要承认一件事：遗忘是一种能力，不是缺陷。你需要主动淘汰过时结论、降权低置信信息、提升冲突证据优先级。更底层地看，记忆不是为了“记住发生过什么”，而是为了“提升下一步决策质量”。所以每条长期记忆都应带来源、时间、置信度和适用条件。只有这样，系统才能在新证据到来时进行信念更新，而不是机械复读旧答案。稳定的 Agent，靠的不是强记忆，而是可修正记忆。

最后给一个可执行标尺：把这套方法交给一个不在场的同事，他能否仅凭日志、配置和文档，在一小时内复盘决策并复现实验结果。如果做不到，说明系统仍依赖个人经验而不是工程能力。继续把隐性判断显式化，直到可测、可审计、可回滚。
