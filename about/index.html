关于我

你好，我是 YeeFay 👋

研究生在读目前，全部精力投在 Agent 方向，目前主力是 Python 和 TypeScript，为啥是这两个：一个是 Python 做 Agent 的生态良好有很多现成的包，另一个为啥用 TS 因为 Agent 主要是编排工作流，本质是一个 I/O 流的事情，CC、Openclaw 等采用了 TS，这一块有很多现成的技术可以参考，因为 CC 和 Openclaw 的源码是现成的，可以学习到很多工程思想。

日常写代码用 Claude Code 和 Codex，主力是 CC 基本上，目前体感 CC 的整个工具调用以及整个的 harness 非常符合我对 Harness 工程的幻想。

做事看长期价值，第一性原理，批判性思维，以及分辨出什么是对的事如何把事情做对。

AI Native 思维要真落到日常，凡是能用 AI 帮忙提效解决的我都会用 AI，能复用的我就搭建 skill。

职业方向很明确：深耕 Agent，主要是工程化，像模型后训练只是我有别的精力会去了解，然后感受现在前沿模型的能力。

我觉得未来的 Agent 一定是未来的主流！能用代码限制的一定不用提示词，保持上下文的整洁！上下文会变的能简洁的就简洁该加重的适当加重，真正的做到平衡，真实的发挥模型的能力！我觉得随着模型能力的增强 Agent 大有可为，一个好的 Agent 才能发挥出模型的百分百的实力！

🔭 在研究什么

主要在做 AI Agent 智能体系统，关注几个具体问题：

  • Agent 架构怎么设计，才真实的有用可持续
  • 记忆系统怎么做，才能真正的发挥上下文的全部实力
  • Agent 怎么准确识别意图，少做无用功
  • 真正有效的 Agent 评测体系长什么样
  • Harness 怎么搭，什么才是好的 Harness
